# RFM Customer Segmentation & Automated Reporting Agent

Teaching materials for a 2-hour NUS guest lecture (*Data Analytics in Business*,
Year-2 elective): an AI agent that takes raw retail transactions and fully
automates **data cleaning → RFM segmentation → marketing strategy report**.

## Contents

| Path | What it is |
|---|---|
| `nus_lecture_rfm_agent.pptx` | 2-hour lecture slide deck (generated by `build_ppt.py`) |
| `app.py` | Streamlit demo UI — run this live in class |
| `agent/` | The agent pipeline (cleaner → RFM engine → strategist → report) |
| `data/retail_transactions.csv` | Sample *dirty* dataset (5.5k rows, 800 customers) |
| `data/generate_dataset.py` | Regenerates the sample dataset |
| `build_ppt.py` | Regenerates the slide deck |
| `build_abstract.py` | Regenerates `nus_lecture_abstract.docx` (bio + abstract for professors) |

## Quick start

```bash
cd ~/PATH/TO/YOUR/CODE
pip install -r requirements.txt
streamlit run app.py
```

Then press **▶ Run agent** in the sidebar. The agent narrates each step
(cleaning log, scoring, segmentation, strategy) before showing the dashboards
and the downloadable report.

### Optional: LLM strategist mode

```bash
export ANTHROPIC_API_KEY=sk-ant-...
streamlit run app.py
```

Toggle **Use Claude as AI strategist** in the sidebar — Claude (claude-opus-4-8)
writes a tailored strategy narrative per segment, referencing the actual
numbers. Without a key the agent uses its built-in rule playbook, so the demo
always works offline.

## The four segments

Scores R / F / M are quintiles (1–5); "high" = strictly above the population
mean score.

| Segment | Recency | Frequency | Monetary | Play |
|---|---|---|---|---|
| High Value Customer | high | high | high | Retain & grow (VIP, early access) |
| Important Customer | high | low | high | Build frequency (cross-sell, habit loops) |
| Important Maintaining Customer | low | high | high | Win back before churn |
| Normal Customer | — | — | — | Low-cost nurture at scale |

## Using your own data

Upload any CSV with at least: `customer_id`, `order_id`, `order_date`,
`amount`. The cleaner handles duplicates, missing keys, mixed date formats,
currency symbols, refunds, and outliers automatically.
